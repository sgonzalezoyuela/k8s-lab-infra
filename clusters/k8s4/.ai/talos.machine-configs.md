# Companion: talos.machine-configs

## Why split base + patches
`talosctl gen config` produces cluster-wide PKI + secrets we want to generate ONCE
and keep. Per-node differences (hostname, IP, install disk) belong in patches
that we can re-render any time `.env` changes.

## Layout

```
talos/
  patches/
    cp.yaml.tpl           # envsubst template
    wk0.yaml.tpl
  scripts/
    gen-config.sh         # invoked by `just talos-config`
_out/
  controlplane.yaml       # generated by `talosctl gen config`
  worker.yaml             # generated by `talosctl gen config`
  secrets.yaml            # ditto, NEVER regenerate after first run
  patches/cp.yaml         # rendered from cp.yaml.tpl
  patches/wk0.yaml        # rendered from wk0.yaml.tpl
  cp.yaml                 # final = controlplane.yaml + patches/cp.yaml
  wk0.yaml                # final = worker.yaml + patches/wk0.yaml
  talosconfig             # patched with endpoints+nodes
```

## `talos/patches/cp.yaml.tpl`

```yaml
machine:
  network:
    hostname: ${CP_HOSTNAME}
    nameservers:
      - ${NETWORK_DNS}
    interfaces:
      - deviceSelector:
          physical: true
        addresses:
          - ${CP_IP}/${NETWORK_CIDR}
        routes:
          - network: 0.0.0.0/0
            gateway: ${NETWORK_GATEWAY}
  install:
    disk: /dev/sda
```

`talos/patches/wk0.yaml.tpl` is identical structure with `${WK0_HOSTNAME}` and
`${WK0_IP}`.

## `talos/scripts/gen-config.sh` (sequence)

1. **Generate base configs once**. If `_out/secrets.yaml` is missing:
   ```
   talosctl gen secrets -o _out/secrets.yaml
   talosctl gen config "${CLUSTER_NAME}" "https://${CP_IP}:6443" \
     --output-dir _out --with-secrets _out/secrets.yaml --force
   ```
   If `_out/secrets.yaml` exists, only re-render base configs (no `gen secrets`)
   so PKI is preserved across reruns.

2. **Render patches** from templates:
   ```
   envsubst < talos/patches/cp.yaml.tpl  > _out/patches/cp.yaml
   envsubst < talos/patches/wk0.yaml.tpl > _out/patches/wk0.yaml
   ```

3. **Patch base configs** to produce final per-node configs:
   ```
   talosctl machineconfig patch _out/controlplane.yaml \
     --patch @_out/patches/cp.yaml -o _out/cp.yaml
   talosctl machineconfig patch _out/worker.yaml \
     --patch @_out/patches/wk0.yaml -o _out/wk0.yaml
   ```

4. **Validate**:
   ```
   talosctl validate --config _out/cp.yaml  --mode metal
   talosctl validate --config _out/wk0.yaml --mode metal
   ```

5. **Patch talosconfig** with endpoints and nodes:
   ```
   talosctl --talosconfig _out/talosconfig config endpoint ${CP_IP}
   talosctl --talosconfig _out/talosconfig config node     ${CP_IP} ${WK0_IP}
   ```

## Justfile addition

```just
talos-config: env-check
    ./talos/scripts/gen-config.sh
```

## Why deviceSelector instead of interface name
Talos can't predict whether the NIC inside the VM is `eth0`, `ens18`, or
something else. `deviceSelector.physical: true` matches the first physical NIC,
which works for single-NIC VMs.

## Failure modes
- `talosctl validate` reports schema errors → patch YAML is malformed; check
  envsubst output (`_out/patches/<node>.yaml`) by eye.
- `gen config` complains "secrets already exist" → expected on rerun; the
  script must use `--force` (regenerate base, keep secrets).
